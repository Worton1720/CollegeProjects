# Recipe Manager

Веб-приложение для удобного хранения, управления и поиска кулинарных рецептов.

## Возможности приложения

**Recipe Manager** предоставляет полный набор функций для работы с рецептами:

- Создание рецептов с детальным описанием (ингредиенты, инструкции, время приготовления, категория)
- Просмотр полного списка рецептов в удобном формате карточек
- Фильтрация по категориям (Супы, Салаты, Выпечка и др.)
- Детальный просмотр каждого рецепта
- Редактирование и удаление существующих рецептов

## Технологии

### Серверная часть
- **Node.js + Express** — REST API
- **SQLite** — файловая база данных
- **CORS** — поддержка кросс-доменных запросов

### Клиентская часть
- **React** — интерфейс пользователя
- **React Router** — клиентская маршрутизация
- **Styled Components** — стилизация
- **Context API** — глобальное состояние

## Структура базы данных

### Таблица `recipes`
```
id (INTEGER PRIMARY KEY AUTOINCREMENT)
title (TEXT NOT NULL) — название рецепта
ingredients (TEXT NOT NULL) — ингредиенты
instructions (TEXT NOT NULL) — пошаговая инструкция
cookingTime (INTEGER) — время приготовления (мин)
category (TEXT) — категория рецепта
createdAt (DATETIME) — дата создания
updatedAt (DATETIME) — дата обновления
```

## API endpoints

| Метод | Эндпоинт | Описание | Параметры |
|-------|----------|----------|-----------|
| `GET` | `/api/recipes` | Список всех рецептов | `?category=` |
| `GET` | `/api/recipes/:id` | Один рецепт | - |
| `POST` | `/api/recipes` | Создать рецепт | `{title, ingredients, instructions, cookingTime?, category?}` |
| `PATCH` | `/api/recipes/:id` | Обновить рецепт | Частичные данные |
| `DELETE` | `/api/recipes/:id` | Удалить рецепт | - |
| `GET` | `/api/categories` | Список категорий | - |

## Серверная логика

1. **Валидация** — проверка обязательных полей
2. **CRUD** — все операции с базой данных
3. **Фильтрация** — по категориям через query-параметры
4. **Обработка ошибок** — стандартизированные ответы

## Клиентские страницы

- **Главная** `/` — список рецептов + фильтр
- **Добавить** `/add` — форма создания
- **Детали** `/recipe/:id` — полный просмотр
- **Редактировать** `/edit/:id` — форма обновления

## Ключевые компоненты

- `Header` — навигация
- `RecipeCard` — карточка рецепта
- `CategoryFilter` — фильтр категорий
- `RecipeContext` — состояние приложения
