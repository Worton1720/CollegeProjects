# Инструкция по установке и запуску проекта "Менеджер рецептов"

## Шаг 2: Установка и запуск Backend

### 2.1 Переходим в папку backend

```bash
cd backend
```

### 2.2 Устанавливаем зависимости

```bash
npm install
```

Это установит:

-   express
-   sqlite3
-   cors

### 2.3 Инициализируем базу данных

```bash
npm run init-db
```

Эта команда:

-   Создаст файл `recipes.db`
-   Создаст таблицу `recipes`
-   Добавит тестовые данные (3 рецепта)

Вы увидите сообщения:

```
✅ Подключение к базе данных установлено
✅ Таблица recipes создана успешно
✅ Тестовые данные добавлены
✅ База данных инициализирована и закрыта
🎉 Инициализация базы данных завершена успешно!
```

### 2.4 (Опционально) Тестируем модуль работы с файлами

```bash
npm run test-file
```

Эта команда протестирует модуль `fileOperations.js`:

-   Запишет данные в файл `recipes_data.json`
-   Прочитает их обратно
-   Выведет результат в консоль

### 2.5 Запускаем сервер

```bash
npm start
```

Сервер запустится на порту **3001**. Вы увидите:

```
✅ Сервер подключен к базе данных
🚀 Сервер запущен на порту 3001
📍 API доступен по адресу: http://localhost:3001
📚 Документация: http://localhost:3001/
```

**Оставьте этот терминал открытым!** Сервер должен работать постоянно.

### 2.6 Проверка работы API

Откройте браузер и перейдите по адресу:

```
http://localhost:3001/
```

Вы увидите информацию об API. Также можете протестировать:

```
http://localhost:3001/api/recipes
```

## Шаг 3: Установка и запуск Frontend

### 3.1 Открываем НОВЫЙ терминал

Не закрывайте терминал с backend! Откройте новое окно терминала.

### 3.2 Переходим в папку frontend

```bash
cd recipe-manager/frontend
```

### 3.3 Устанавливаем зависимости

```bash
npm install
```

Это установит:

-   react
-   react-dom
-   react-router-dom
-   styled-components
-   react-scripts

**Примечание:** Установка может занять несколько минут.

### 3.4 Запускаем приложение

```bash
npm start
```

Приложение автоматически откроется в браузере по адресу:

```
http://localhost:3000
```

Вы увидите главную страницу с тремя тестовыми рецептами!

## Шаг 4: Проверка работы приложения

### 4.1 Главная страница

-   Вы должны увидеть 3 рецепта: Борщ, Паста Карбонара, Цезарь салат
-   Над списком есть фильтр по категориям
-   Попробуйте отфильтровать по категории "Супы"

### 4.2 Просмотр рецепта

-   Нажмите на любую карточку рецепта
-   Откроется страница с детальной информацией
-   Вы увидите полные ингредиенты и инструкции

### 4.3 Добавление рецепта

-   Нажмите "Добавить рецепт" в навигации
-   Заполните форму
-   Нажмите "Добавить рецепт"
-   Вы будете перенаправлены на главную страницу, где увидите новый рецепт

### 4.4 Редактирование рецепта

-   Откройте любой рецепт
-   Нажмите кнопку "Редактировать"
-   Измените данные
-   Нажмите "Сохранить изменения"

### 4.5 Удаление рецепта

-   Откройте любой рецепт
-   Нажмите кнопку "Удалить"
-   Подтвердите удаление
-   Рецепт будет удален из списка

## Структура запущенного проекта

После запуска у вас будут работать:

**Backend (порт 3001):**

-   ✅ Express сервер
-   ✅ SQLite база данных
-   ✅ REST API для работы с рецептами

**Frontend (порт 3000):**

-   ✅ React приложение
-   ✅ Роутинг через React Router
-   ✅ Стилизация через Styled Components
-   ✅ Глобальное состояние через Context API

